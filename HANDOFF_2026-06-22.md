## 4. 在新电脑恢复项目的最小步骤

```bash
git clone https://github.com/ljx418/interview_service.git
cd interview_service

python3 -m venv .venv
source .venv/bin/activate
pip install -e .

npm --prefix apps/chatbox install

python3 -m pytest
npm --prefix apps/chatbox run build
```

启动后端：

```bash
python3 -m uvicorn services.api.main:app --host 127.0.0.1 --port 8000
```

启动前端：

```bash
npm --prefix apps/chatbox run dev -- --host 127.0.0.1 --port 5173
```

打开：

```text
http://127.0.0.1:5173
```

如需重采集 P4B 截图，先启动 Chrome remote debugging，再运行：

```bash
CHROME_DEBUG_PORT=9222 node scripts/capture_p4_workbench_evidence.mjs
```

## 5. 密钥与外部 provider

仓库不应包含 `.env` 或 API Key。默认模式是 mock provider，无需密钥即可完成本地验收。

如果需要在新电脑启用 MiniMax 或 OpenAI-compatible provider，请复制：

```bash
cp .env.example .env
```

然后填写本地 `.env`。不要把 API Key 提交到 git。
