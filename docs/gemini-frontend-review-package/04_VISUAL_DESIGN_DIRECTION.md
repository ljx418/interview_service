# 视觉设计方向

## 设计气质

JobPilot AI 应像一个安静、可靠、为求职材料服务的工作台，不应像营销页、聊天玩具或复杂 BI dashboard。

关键词：

- 本地优先；
- 求职材料；
- 可信；
- 可确认；
- 可导出；
- 轻量但不简陋；
- 专业但不压迫。

## 色彩方向

建议使用中性背景加少量功能色：

- 背景：`#f6f8fb` / `#ffffff`
- 主文字：`#111827`
- 次文字：`#5b677a`
- 边框：`#d8e0ea`
- 主操作蓝：`#2457f5`
- 成功绿：`#15803d`
- 警告琥珀：`#b45309`
- 错误红：`#b42318`

避免：

- 大面积单一蓝紫渐变；
- 装饰性光球；
- 营销 hero；
- 过多彩色状态标签。

## 字体与层级

建议：

- 页面标题：28-34px；
- 面板标题：18-22px；
- 卡片标题：15-17px；
- 正文：14-16px；
- 辅助信息：12-13px。

规则：

- 不使用 viewport width 缩放字体；
- 字间距保持 0；
- 面板内部不要使用 hero 级大字；
- 长词、中文长句、按钮文字必须换行或收缩，不得溢出。

## 圆角与密度

- 卡片圆角不超过 8px；
- 按钮圆角不超过 8px；
- 状态标签可以轻微圆角，但不使用夸张胶囊；
- 页面应有工作台密度，不做大面积空洞。

## 图标

推荐使用图标表达常见操作：

- 上传；
- 发送；
- 导出；
- 编辑；
- 重新生成；
- 展开来源；
- 警告。

图标按钮必须有 tooltip 或可访问名称。

## 操作层级

产物卡必须区分主次操作：

- Primary action：补充事实、去确认、导出当前版本；
- Secondary action：查看详情、编辑、重新生成、查看来源；
- Destructive 或高风险动作不得与普通按钮同等样式。

阻塞状态建议使用轻量琥珀色边框和清晰主按钮，不使用大面积警告色。

## 状态语言

状态文案应面向用户：

- 推荐：`外部模型未调用（隐私安全）`
- 推荐：`外部调用需要确认`
- 推荐：`正在对比 JD 与你的项目经历`
- 避免：`External configured`
- 避免：`provider ready`
- 避免：`artifact validation failed`

待确认项应使用求职辅导语气：

- 推荐：`为了让简历更具说服力，建议补充性能提升的具体指标。`
- 避免：`性能提升 30% 是否有证据？`
