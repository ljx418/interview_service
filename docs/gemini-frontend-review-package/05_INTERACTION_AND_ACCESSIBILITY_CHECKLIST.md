# 交互与可访问性清单

## 首屏

- [ ] 5 秒内能看出第一步；
- [ ] Chatbox 空状态 suggested prompts 比工程状态更突出；
- [ ] suggested prompt 点击后能填入 composer 或触发对话；
- [ ] 示例模式和我的资料模式含义清楚；
- [ ] provider 是否外呼没有歧义；
- [ ] 没有大面积无意义空白。

## 对话

- [ ] 发送有效任务有响应；
- [ ] 缺资料有下一步；
- [ ] 后端错误有恢复动作；
- [ ] loading / thinking 状态说明正在做什么；
- [ ] 长内容可折叠；
- [ ] Enter / Shift+Enter 行为清楚。

## 推进台

- [ ] 用户能区分对话输入和结果管理；
- [ ] 当前任务、下一步和产物状态可扫读；
- [ ] 待确认项明显；
- [ ] 版本信息存在但不喧宾夺主；
- [ ] 导出受阻时说明原因。
- [ ] 初始状态说明产物将在导入资料后生成。

## 产物卡

- [ ] 使用求职语义而不是工程术语；
- [ ] 不展开 JSON 也能理解；
- [ ] source refs 可查看；
- [ ] questions_to_confirm 可查看；
- [ ] 主要操作优先级清楚。
- [ ] primary action 和 secondary action 视觉层级清楚。
- [ ] 待确认项使用求职辅导语气。

## 响应式

- [ ] 1280px 双栏清晰；
- [ ] 720px 堆叠后不出现半屏空洞；
- [ ] 390px 没有严重横向滚动；
- [ ] 输入区不遮挡消息；
- [ ] 390px 下 Workbench 不挤压 Chatbox；
- [ ] 主要按钮不溢出。

## 可访问性冒烟

- [ ] 按钮有可理解文本或 aria-label；
- [ ] segmented mode toggle 有 `aria-pressed` 或等价状态；
- [ ] 焦点状态可见；
- [ ] 文本对比度达到 WCAG AA；
- [ ] 表单 label 或说明清楚；
- [ ] 错误状态不仅靠颜色表达；
- [ ] 动效不影响理解。

## 虚假验收风险

- [ ] 静态原型没有被写成已实现；
- [ ] Gemini 审查没有被写成人工验收通过；
- [ ] examples 数据没有被写成真实个人资料；
- [ ] 外部 provider 未调用时没有写成真实外部模型验收；
- [ ] 截图必须来自真实 Chrome。
