# P9 图像生成提示词与审查规则

本阶段优先使用受控 SVG 原型图，避免 AI 文生图生成与项目无关的 SaaS 页面。若后续使用宿主 imag2 模型，必须按以下提示词和审查规则执行。

## 1. 概念图提示词

```text
生成一张中文产品架构概念图，主题是 JobPilot AI Chatbox-native 求职情报与申请包工作台。
画面包含四个清晰区域：顶部服务中心、左侧求职态势图、中央 Chatbox 主控台、右侧产物台。
使用产品架构图风格，不要营销海报，不要人物插画。
用颜色区分：已实现基线、P9 待开发、后续高风险、禁止默认实现。
不要出现招聘平台接入、自动投递、真实 provider 验收通过等结论性字样。
```

## 2. 目标页面图提示词

```text
基于 JobPilot AI 当前 Chatbox 工作台，生成一张 P9 目标前端页面图。
页面为高端、克制、专业的工作台，而不是营销落地页。
顶部是服务状态栏：Provider、MCP、Skill、ASR、搜索源、Workspace、诊断。
左侧是可折叠求职态势图：城市薪资、技术栈热度、投递流程。
中央是最重要的 Chatbox：状态机、消息时间线、输入框。
右侧是产物台：目标 JD、事实摘要、简历草稿、source refs、待确认项、导出预检。
不要大量向导卡片，不要让表单压住聊天。
所有未实现能力标注为目标设计，不代表已实现。
```

## 3. 用户路线图提示词

```text
生成一张中文用户路线图，展示 JobPilot AI P9 的端到端流程：
搜索 JD、汇总薪资城市、选择目标岗位、补全资料、生成申请包、调整事实、更新投递流程。
每一步包含：用户输入、系统动作、可见产物、验收证据。
风格为清晰的产品流程图，不要漫画，不要营销插画。
```

## 4. AI 图像审查规则

生成图必须逐项审查：

- 是否保留中央 Chatbox 主控。
- 是否包含顶部服务中心、左侧态势图、右侧产物台。
- 是否错误声称招聘平台、真实 provider、ASR、MCP、Skill 已通过。
- 是否出现与 JobPilot 无关的 CRM、销售、财务或通用 SaaS 语义。
- 是否用大量向导卡片替代 Chatbox。
- 是否能指导后续自动化开发和验收。
