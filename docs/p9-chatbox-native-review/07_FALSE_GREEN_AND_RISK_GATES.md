# P9 False-Green 与风险门槛

## 1. 禁止写成已通过的能力

- 全网 JD 搜索真实接入。
- BOSS、猎聘、拉勾、LinkedIn 等招聘平台登录或自动抓取。
- 自动沟通招聘方。
- 自动投递。
- 真实 LLM provider 质量。
- ASR 真实识别质量。
- MCP / Skill 外部连通性。
- 真实个人资料路径。
- SaaS、多租户、Billing。

## 2. 报告必写边界

任何 P9 文档、原型或验收报告必须写清：

- 当前是文档设计或目标原型，不是代码实现。
- 当前目标图不是已实现截图。
- 当前真实截图只代表 P8.1 基线。
- 公开 JD 搜索仅允许合规来源。
- ASR、真实 provider、搜索 API 均需 opt-in。

## 3. 自动打回条件

出现以下任一情况必须打回：

- 目标图或文档给出招聘平台接入已完成的结论。
- 把目标设计图当成真实截图。
- 把 provider configured 写成 provider called。
- 把 ASR 转写直接写入正式简历。
- 无 source refs 的经历可以正式导出。
- 中央 Chatbox 被大型向导卡片压住。
- 左侧图表无法折叠并挤压 Chatbox。
