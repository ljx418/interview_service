# 用户路径与验收门槛

## 必须覆盖的用户路径

1. 首屏打开
   - 用户看到本地就绪、示例模式、外部模型未调用。
   - 用户看到建议任务并能知道第一步。

2. 导入资料
   - 用户点击“导入资料”或上传入口。
   - 系统说明会在本地生成职业事实和技能证据。

3. 粘贴 JD
   - 用户粘贴目标 JD。
   - 系统显示解析计划、匹配缺口和申请包草稿入口。

4. 自由连续对话
   - 用户说“我还没有 JD，先聊聊求职方向和偏好”。
   - 用户继续问“下一步先补 React 还是项目经历”。
   - 系统给建议，不生成 artifact。

5. 状态查询
   - 用户问“当前进展如何，有哪些产物”。
   - 系统基于 workspace 返回状态摘要。

6. 显式工具生成
   - 用户明确说“请整理资料，生成职业事实”。
   - 系统生成职业事实产物卡，并让 Workbench 主体与产物计数保持一致。

7. 示例路径完成
   - 示例数据完成 9/9 步。
   - Workbench 展示职业事实、项目卡、岗位解析、匹配报告、申请包、导出文件和待确认项。

8. 错误恢复
   - 缺资料、缺 JD 或请求失败时，显示原因和可执行恢复动作。

9. 移动端恢复
   - 390px 下 Chatbox 优先，Workbench 作为底部抽屉或次级入口。

## P4 验收门槛

- 5 秒内知道第一步。
- 对话区、任务启动、Workbench、产物卡职责清楚。
- provider 和隐私状态没有歧义。
- 普通自由追问不误触发 artifact 写入。
- 明确工具意图能生成对应产物。
- 1200/1440/1600/1920 桌面不出现大面积空白。
- 720 窄屏和 390 移动端核心输入与推进台入口可达。
- 报告和原型不声称人工验收、真实个人资料或真实外部 provider 已通过。
