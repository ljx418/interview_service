# P9 外部审计意见修订记录

状态：文档修订阶段。本文档记录 ChatGPT 外部审计意见的采纳结果，不代表 P9 UI、真实 JD 搜索、ASR、MCP/Skill、真实 provider、招聘平台自动接入或自动投递已经实现。

## 1. 外部意见摘要

外部审计认可当前 P9 文档体系可以支撑下一阶段自动化开发，但要求增加一个关键限定：

```text
P9 可以进入自动化开发
但必须严格限制在 Chatbox-native 工作台重构 + 求职情报可视化层
不得提前进入真实数据接入、ASR、MCP/Skill、自动投递等能力实现
```

外部审计进一步指出，P9 的本质不是新招聘/搜索/语音/平台系统，而是：

```text
Chatbox-first interaction layer
+ 左侧求职态势图（可视化信息层）
+ 右侧产物台（Artifact workspace）
+ 顶部服务状态（control-plane visibility）
```

## 2. 已采纳的关键约束

本轮修订将以下约束写入 PRD、目标架构、验收门槛、详细计划、覆盖度复审和 drawio 文本镜像：

1. 不新增真实数据源系统。
2. 不新增外部平台接入。
3. 不新增长期运行任务系统。
4. 不新增独立业务服务。
5. P9 只允许 UI 重排、现有 API 组合、Artifact 展示增强和 Chatbox 交互增强。

这些约束的目的不是降低 P9 的目标体验，而是防止后续自动化开发把 P9 扩张成 BI 平台、招聘平台爬虫、语音系统、MCP/Skill 平台或自动投递系统。

## 3. 修订后的 P9 定位

P9 的准确定位：

```text
Chatbox-first UI 与信息结构重构
+ 求职情报可视化层
+ 现有 JobPilot 能力重新组织
```

P9 不做：

- 真实 JD 搜索系统；
- 招聘平台账号登录、抓取、绕风控或自动沟通；
- 真实 ASR 系统；
- MCP/Skill 平台化；
- 自动投递；
- 新独立后端服务；
- 长期运行调度任务。

## 4. 对各阶段的修订影响

| 阶段 | 修订后边界 |
| --- | --- |
| P9-M1 | 只做 Chatbox-native 信息架构重排，不重写业务系统 |
| P9-M2 | 顶部服务中心只展示状态和配置边界，不执行真实接入 |
| P9-M3 | 左侧态势图是可视化信息层，不是 BI 分析引擎 |
| P9-M4 | JD source run 只能使用用户粘贴、fixture、已有本地/公开示例数据，不做真实平台接入或长期任务 |
| P9-M5 | ASR 只允许展示 opt-in 状态或文本引导；真实麦克风和外部语音服务延期 |
| P9-M6 | 多 JD 申请包复用现有 P8/P5.5 能力和 Artifact 机制 |
| P9-M7 | 流程更新只改本地 workspace/artifact 状态，不对外沟通或投递 |
| P9-M8/M9 | 验收重点是 UI、可视化、Chatbox 交互和证据边界 |

## 5. 修订结论

外部审计意见已采纳。P9 文档仍可进入 P9-M0，但进入开发时必须按“UI 信息架构重构 + 求职情报可视化层”的限定执行。

允许声明：

- P9 文档可以支撑下一阶段自动化开发；
- P9 开发范围已经被约束为 UI/信息结构/现有能力重组；
- P9-M0 可作为开发前启动审计入口。

不允许声明：

- P9 是真实 JD 搜索系统；
- P9 已实现真实 ASR；
- P9 已接入 MCP/Skill 平台；
- P9 已接入招聘平台；
- P9 已实现自动投递；
- P9 已通过真实 provider 或真实个人资料路径验收。
