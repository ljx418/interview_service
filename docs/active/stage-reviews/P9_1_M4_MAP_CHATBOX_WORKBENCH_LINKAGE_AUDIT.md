# P9.1-M4 地图、Chatbox、产物台联动审计

状态：实现候选已落地；最终通过状态以 P9.1-M5 全量自动化报告、截图和 eval 为准。

## 开发内容

- 行政区划地图点击区域后可通过 Chatbox 追问该区域的岗位、薪资、技术栈或来源；
- 图层切换和区域详情让用户理解“岗位数量、薪资中位、技术热度、来源可信度”的差异；
- 右侧产物台保留简历草稿、事实摘要、待确认项和不可声明清单；
- Chatbox 仍是中心主路径，地图和产物台是辅助信息层。

## PRD 规格检视

| PRD 要求 | 实现候选 |
| --- | --- |
| 用户可通过 Chatbox 更新招聘态势信息 | 地图区域与岗位问题可进入 Chatbox 追问 |
| 左侧态势图可折叠/切换 | 保留 market/match/pipeline 页签结构 |
| 右侧产物台持续可见事实和交付物 | P9ArtifactOverview 展示 Socratic facts 和确认项 |
| 不越界做真实平台自动化 | 没有平台登录、抓取、自动沟通或投递实现 |

## 验收证据要求

- 截图覆盖市场地图、Chatbox Socratic 对话、右侧产物台；
- 自动化步骤能从地图进入城市下钻，再启动 Chatbox 资料补全；
- 报告列出当前架构实现与目标架构差异。

## 打回条件

- 地图成为中心主路径并压住 Chatbox；
- Chatbox 和右侧产物台断联；
- 产物台展示编造事实或缺少 pending confirmations。
