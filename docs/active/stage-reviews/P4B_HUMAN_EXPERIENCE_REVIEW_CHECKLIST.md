# P4B 人工体验审查清单

日期：2026-06-25
阶段：P4B 自动化完成后 / 人工体验审查门槛

## 1. 审查口径

本清单用于把人工主观体验反馈转成可执行的 P4C backlog 或 P4 冻结结论。它不替代自动化验收报告，也不能把 examples 数据写成真实个人资料验收。

当前可审查范围：

- 本地 Chatbox 页面体验；
- `docs/reports/P4B_AUTOMATED_ACCEPTANCE_REPORT.html`；
- 1200px、1440px、1600px、1920px、720px、390px 截图证据；
- examples 匿名真实感数据路径；
- mock provider 默认路径。

当前不可审查为通过的范围：

- 真实个人简历、真实私有 JD 或真实 transcript；
- 真实外部 provider 默认生产路径；
- MCP、CLI、ASR、会议平台、自动投递、SaaS 登录、多租户或 Billing。

## 2. 审查步骤

1. 打开自动化报告：

```text
docs/reports/P4B_AUTOMATED_ACCEPTANCE_REPORT.html
```

2. 启动本地服务：

```bash
python3 -m uvicorn services.api.main:app --host 127.0.0.1 --port 8000
npm --prefix apps/chatbox run dev -- --host 127.0.0.1 --port 5173
```

3. 打开 Chatbox：

```text
http://127.0.0.1:5173
```

4. 按以下路径人工操作：

```text
首屏理解下一步
→ 点击 suggested prompt
→ 运行示例路径
→ 查看推进台产物
→ 查看待确认项、版本、导出
→ 切换或模拟 1200 / 1440 / 1600 / 1920 / 720 / 390 宽度
→ 判断桌面、窄屏、移动端是否顺手
```

## 3. 评分表

每项使用 0-2 分：

- 0 = 不通过，必须进入 P4C；
- 1 = 可接受但需要微调；
- 2 = 认可，可作为 P4 冻结证据。

| 项目 | 分数 | 记录 |
| --- | --- | --- |
| 5 秒内能理解第一步 | 2 | 人工反馈：体验 OK，可进入冻结操作。 |
| Suggested prompts 与对话输入闭环自然 | 2 | 快捷动作已统一到输入框上方，并能填入或触发对话。 |
| 发送任务后能理解系统正在做什么 | 2 | 对话区已增加 Agent 状态机、进度和下一步说明。 |
| 缺资料或错误时恢复路径明确 | 2 | 自动化报告和当前 UI 保留补充 JD、上传资料、示例路径等恢复入口。 |
| Workbench 不抢占对话但能解释当前进度 | 2 | 桌面右侧推进台聚焦产物；移动端推进台入口仅在有产物后出现。 |
| 产物卡不依赖 JSON 也能理解价值 | 2 | 产物卡展示岗位、匹配、申请包、面试准备等求职语义摘要。 |
| 待确认项文案像求职辅导而非程序校验 | 2 | 待确认项保留为导出前风险和补充事实提示。 |
| 1200px 桌面像完整工作台 | 2 | P4/P4B 自动化截图覆盖桌面三栏工作台；人工体验认可。 |
| 1440/1600/1920px 没有大面积无意义空白 | 2 | 全尺寸桌面 hard gate 自动化证据已完成；人工体验认可。 |
| 720px 窄屏操作顺手 | 2 | 窄屏截图与布局修正已完成。 |
| 390px 移动端主要任务可达 | 2 | 输入区固定可用，首屏无遮挡；推进台不再压住初始聊天。 |
| provider / 隐私 / 外呼语义不误导 | 2 | 默认 mock、本地、不外呼语义清晰；真实 provider 仍需显式确认。 |
| 视觉质感达到可长期使用的工具感 | 2 | 经本轮 UI 质感和响应式修正后，人工反馈为 OK。 |

建议判定：

- 任一关键项为 0：进入 P4C 修正；
- 总分低于 20：进入 P4C 修正；
- 总分 20-24 且无关键项为 0：可做小幅 P4C polish；
- 总分 25 及以上：可进入 P4 冻结评审。

关键项：

- 5 秒内能理解第一步；
- 发送任务后能理解系统正在做什么；
- 1200px 桌面像完整工作台；
- provider / 隐私 / 外呼语义不误导。

P4C 优先级建议：

- P0：影响第一步理解、任务发送反馈、外呼误导、桌面大面积空白、移动端核心操作不可达；
- P1：影响长期使用意愿、Workbench 信息密度、产物卡阅读效率、错误恢复清晰度；
- P2：视觉 polish、文案微调、次级详情展开方式。
