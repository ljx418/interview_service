本页用于在进入真实资料验收前，先让人类审核“简历 + 背景资料 + 目标 JD + 待确认问题”的资料结构是否足够支撑 P5 真实资料本地闭环。页面内人物、公司、联系方式、项目经历均为合成示例，不代表真实个人资料验收通过。

阶段：P5-REAL 前置资料审核 模式：本地 / mock / 无外部 provider 日期：2026-06-26 状态：待人工审核资料形态

1. 审核边界

重要边界：本页不包含真实 API Key、不调用真实外部模型、不搜索用户电脑中的个人文件、不代表真实个人资料路径已通过。真实资料验收必须由用户提供明确本地路径、允许展示字段和脱敏范围后才能执行。

本页要确认什么

  • JobPilot 需要哪些资料才能生成可信申请包。
  • 哪些字段会进入解析、待确认和导出流程。
  • 哪些缺口必须在真实验收前补齐。

本页不证明什么

  • 不证明真实个人资料已完成验收。
  • 不证明 MiniMax、DeepSeek 或其他 provider 可用。
  • 不证明 P5 已冻结或产品化完成。

建议审核结果

  • 确认合成资料结构是否贴近真实求职材料。
  • 指出需要新增或删减的背景资料项。
  • 决定真实资料验收时允许展示的字段。

导入资料用户上传或粘贴简历、项目说明、经历材料。

编辑导出编辑后重新阻塞，确认后导出 Markdown/DOCX。
期望验收体验：用户能清楚知道系统正在处理什么、哪些事实缺口阻塞导出、为什么某些内容不能直接生成，以及导出的申请包依据哪些本地资料和 JD。

7. 待人工审核问题

级别问题验收影响
Blocking 真实资料验收时，用户是否提供明确本地路径和允许展示字段？ 未确认则不得执行 P5-REAL，不得声称真实资料通过。
Blocking 简历、项目 README、JD 三类资料是否齐全？ 缺任一类都会导致申请包生成依据不足。
Warning 是否允许在报告中展示姓名、项目名、局部原文片段？ 影响报告可读性和隐私风险，需要记录到验收结论。
Warning 项目经历是否有可验证证据，是否存在夸大风险？ 影响申请包可信度，必要时生成“待确认事实”。
Optional 是否需要覆盖第二个目标 JD 或多个城市偏好？ 可作为 P5 后续增强，不阻塞默认闭环。
