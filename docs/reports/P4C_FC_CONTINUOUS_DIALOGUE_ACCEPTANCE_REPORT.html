目标架构

  • Chatbox UI stays a thin browser client for conversation, uploads, mode/provider status, and artifact display.
  • Chat Session State stores user and assistant messages for restore after refresh.
  • Chat Intent Router separates free local dialogue, status/next-step replies, and explicit domain tool intents.
  • Domain Tools write artifacts only for explicit profile, JD, package, or interview requests.
  • Provider-backed free chat remains a later opt-in path and is not used by this default local/mock acceptance.

当前实现

  • React/Vite Chatbox initializes a local workspace and chat session through FastAPI.
  • KeywordChatCore handles local deterministic free dialogue and tool routing.
  • Workspace artifacts and chat messages are persisted in the local SQLite workspace.
  • Browser automation uses Chrome DevTools Protocol screenshots and clears viewport emulation at the end.
  • The UI labels the default privacy boundary as local/mock and external model not called.
