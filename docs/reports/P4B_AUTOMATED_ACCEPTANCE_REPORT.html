本轮目标

P4B 的核心目标是修复全尺寸桌面下“左侧窄栏 + 右侧大面积空白”的体验问题。 验收宽度从基础响应式扩展为 1200 / 1280 / 1440 / 1600 / 1920 / 720 / 390。

桌面

全尺寸工作台

1200px 以上显示任务上下文、对话区、推进台三栏，而不是把窄屏 Chatbox 放大。

窄屏

720px 可用

保留主对话区，推进台以抽屉方式查看，不挤压输入和消息。

移动

390px 可达

底部抽屉可访问产物推进台，核心对话和输入不被遮挡。

目标架构与当前实现

目标架构

  • Experience Shell：只承载 workspace、模式、provider 和隐私状态。
  • Conversation Plane：承载任务入口、消息、loading、错误恢复和下一步。
  • Full-size Desktop Workbench Controller：管理 1200px 以上桌面信息密度。
  • Workbench Plane：只管理当前目标、产物、确认项、版本和导出。
  • Evidence Plane：用截图、测试和报告证明范围，不替代人工体验判断。

当前实现

  • 顶部状态条保留“本地就绪 / 示例模式 / 外部模型未调用（隐私安全）”。
  • 桌面端新增任务上下文面板，形成三栏工作台。
  • Chatbox 空状态保留 suggested prompts。
  • 缺少 JD 时显示“补充 JD / 跑示例路径”。
  • 移动端推进台仍为底部抽屉。
