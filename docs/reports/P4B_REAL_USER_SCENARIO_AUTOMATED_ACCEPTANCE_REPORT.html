目标架构

前端体验平面

  • Experience Shell：展示 workspace、模式、provider 和隐私状态。
  • Conversation Plane：承载 suggested prompts、用户输入、loading、错误恢复和结果反馈。
  • Full-size Desktop Workbench Controller：管理 1200px 以上桌面信息密度。
  • Workbench Plane：只展示当前目标、产物、确认项、版本和导出。
  • Responsive Controller：覆盖桌面、窄屏和移动端，不让 Workbench 压缩主对话。

后端 Agent Tool 架构

  • FastAPI routes 提供 workspace、chat、workflow、artifact、export 和 provider API。
  • ChatCore / PiAgent adapter 负责意图和工具计划，不直接写业务数据。
  • Python Domain Tools 负责资料抽取、JD 解析、匹配、申请包、面试准备和复盘。
  • SQLite workspace 保存 artifact、artifact_version、chat_session、provider_invocation 和导出路径。
  • mock provider 是默认验收基线；外部 provider 必须人工确认。
