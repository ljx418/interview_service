# JobPilot AI 关键验收清单 v1.0

本清单用于判断项目是否真的完成“用户实际得到结果”的核心目标，而不是只完成页面或 Prompt。

---

## 1. 总体验收路径

完整链路必须跑通：

```text
导入资料
→ 生成职业事实库 / 技能证据库 / 项目卡
→ 粘贴 JD
→ 生成岗位适合度报告
→ 生成申请包
→ 导出申请材料
→ 生成面试准备包
→ 完成模拟面试
→ 实时提示当前问题
→ 生成面试复盘
→ 给出下一步训练计划
```

---

## 2. 场景 1：导入资料

### 输入

- 一份转行前端简历；
- 一个 TodoPlus 项目 README；
- 用户目标：Junior Frontend Developer。

### 期望输出

- 5 个以上技能；
- 2 个以上项目或经历；
- 每个技能尽量有关联证据；
- 3 个可补强建议；
- 不确定内容被标记。

### 验收项

- [ ] 无需注册登录；
- [ ] 文件保存到本地 workspace；
- [ ] 能生成 CareerFact；
- [ ] 能生成 SkillEvidence；
- [ ] 能生成 TechProject；
- [ ] 输出语气低压力；
- [ ] 用户能修改和确认事实。

---

## 3. 场景 2：分析岗位

### 输入

- Junior Frontend Developer JD；
- 已建立的用户资料。

### 期望输出

- JD 摘要；
- must-have / nice-to-have；
- 技术栈；
- 适合度标签；
- 用户已具备的证据；
- 建议补强点；
- 是否建议申请；
- 1-3 个下一步行动。

### 验收项

- [ ] 不只输出分数；
- [ ] 能说明为什么适合或不适合；
- [ ] 能引用技能证据；
- [ ] 能输出投前建议；
- [ ] 不使用打击性文案。

---

## 4. 场景 3：生成申请包

### 输入

- JD；
- CareerFact；
- SkillEvidence；
- TechProject。

### 期望输出

- 定制简历草稿；
- 项目描述优化版；
- Recruiter Message；
- 待确认事实清单；
- Markdown/PDF 导出文件。

### 验收项

- [ ] 简历内容针对 JD；
- [ ] 不编造项目或经验；
- [ ] 不确定数字被标记；
- [ ] 项目 bullet 能体现技术栈和结果；
- [ ] 用户可以修改；
- [ ] 文件可导出；
- [ ] 申请包保存在 workspace。

---

## 5. 场景 4：面试准备

### 输入

- 已生成申请包；
- 目标岗位 JD。

### 期望输出

- 面试准备包；
- 自我介绍；
- 转行动机；
- 项目深挖问题；
- 技术基础问题；
- 行为面问题；
- 反问问题；
- 故事卡。

### 验收项

- [ ] 面试问题贴合 JD；
- [ ] 故事卡引用真实项目；
- [ ] 自我介绍控制在 90 秒左右；
- [ ] 转行动机不空泛；
- [ ] 项目深挖问题能覆盖技术难点和取舍。

---

## 6. 场景 5：模拟面试

### 输入

- 面试类型：项目深挖；
- 用户回答文本或语音转文字。

### 期望输出

- 面试官提问；
- 追问；
- 回答评分；
- 改进建议；
- 更好的回答结构；
- 下一题或训练任务。

### 验收项

- [ ] 能追问而不是只问固定题；
- [ ] 能指出回答过长/缺少结果/缺少个人贡献；
- [ ] 能推荐项目卡或故事卡；
- [ ] 能给用户下一步训练任务。

---

## 7. 场景 6：实时提示

### 输入

当前面试问题：

```text
讲一个你解决技术难题的经历。
```

### 期望输出

```text
当前问题：讲一个你解决技术难题的经历
问题类型：项目深挖 / 技术难题
建议用：TodoPlus 登录鉴权项目
回答结构：
1. 先说遇到的问题
2. 讲你如何定位原因
3. 讲你的实现方案
4. 讲一个技术取舍
5. 用结果收尾
提醒：不要说 OAuth，如果项目里没有。
```

### 验收项

- [ ] 能识别问题类型；
- [ ] 能推荐真实项目；
- [ ] 能给结构而不是默认代答；
- [ ] 能提醒不真实内容；
- [ ] 能支持暂停/结束/不保存；
- [ ] 延迟在可接受范围内。

---

## 8. 场景 7：面试后复盘

### 输入

- 面试 transcript；
- 用户回答摘要；
- 面试岗位。

### 期望输出

- 问题清单；
- 回答表现；
- 做得好的地方；
- 建议补强的地方；
- 下次训练计划；
- 感谢消息草稿；
- 申请状态更新建议。

### 验收项

- [ ] 能总结问题；
- [ ] 能指出具体改进点；
- [ ] 能生成训练任务；
- [ ] 能更新申请记录；
- [ ] 用户可删除 transcript。

---

## 9. 安全与隐私验收

- [ ] 无注册登录；
- [ ] 数据默认本地保存；
- [ ] 不默认保存原始音频；
- [ ] 不默认保存视频；
- [ ] 用户可删除工作区；
- [ ] API Key 不写入仓库；
- [ ] 不读取 workspace 外文件；
- [ ] 敏感操作需要用户确认；
- [ ] 不提供隐蔽式面试作弊能力。

---

## 10. 开源可用性验收

- [ ] README 清晰；
- [ ] 有 `.env.example`；
- [ ] 有 Docker Compose；
- [ ] 有示例简历；
- [ ] 有示例 JD；
- [ ] 有完整 demo 脚本；
- [ ] 一条命令能启动；
- [ ] 新贡献者能跑通示例闭环。

---

## 11. MVP 通过标准

MVP 通过不是“功能全”，而是满足：

```text
一个转行程序员可以在本地启动项目，上传简历和项目，粘贴一个目标 JD，得到一份可修改申请包，完成一次面试准备，并在模拟/实时问题中得到结构提示和复盘。
```
